Preguntas a responder

¿Por qué es importante hacer un session_unset() y luego un session_destroy()? session_destroy() destroys all of the data associated with the current session. It does not unset any of the global variables associated with the session, or unset the session cookie. To use the session variables again, session_start() has to be called.[1]

¿Cuál es la diferencia entre una variable de sesión y una cookie? A session is a way to store information (in variables) to be used across multiple pages. Unlike a cookie, the information is not stored on the users computer.[2]

¿Qué técnicas se utilizan en sitios como facebook para que el usuario no sobreescriba sus fotos en el sistema de archivos cuando sube una foto con el mismo nombre? A good practice is to make a sha256 of the file and then saving it.
How to do it:

  • Upload the files to a temporary directory.
  • Hash the file and get the sha256 string.
  • Rename the file and copy to upload folder.[3]

¿Qué es CSRF y cómo puede prevenirse? Cross Site Request Forgery and it refers to users who are genuinely logged in or authorised to use a service or website, find their authorisation exploited by another user who has embedded exploitable code on a page that they visit. This doesn’t even have to be on the same site.[4]
