Programación orientada a eventos

¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?
Por ejemplo, sirve para saber si el usuario lleno todos los campos con los datos válidos. No tienes que esperar a que lo valide el servidor. Se detiene el proceso si no es una entrada válida.

¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?
Lo puedes delegar al navegador web o al servidor.

Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?
Es una buena práctica porque aunque el servidor puede hacer la validación, lo que pasará es que rechazará la entrada. Se perdería tiempo en esperar que el servidor responda. Todo esto se puede evitar con tan solo validar con Javascript e impedir el submit.
